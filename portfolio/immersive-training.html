Overview

Support the development of maintenance skills

As a part of a coursework, our team (of four) was provided with an initial goal to develop a highly interactive and informative system for assitance during training. We had to demonstrate how future maintenance workers can benefit from highly interactive Augmented Reality(AR) technology.

Context

WinGD (Winterthur Gas & Diesel Ltd.) approached our department to develope a proof of concept of how the AR technologies can assist their use-case in future. Since WinGD is a leading developer of ship engines, they asked us to build a prototype that lets the user explore the components of a ship-engine valve.

They provided us with the following materials:

  • A physical valve
  • A virtual model of the physical valve (.FBX)
  • Current training materials as PDF documents

Understanding Needs

Defining Goals

After reading through the documents and understanding the procedure surrounding the maintenance of the valve, we came up with three realistic goals. These goals will facilitate as checkpoints for the project and help us to segregate the system into 3 standalone working modules.

1. Exploratory Training
  • Learn about the physical model with the help of “augmented” elements
2. Disassembly & Assembly
  • Follow step-by-step instructions to deconstruct the elements and reconstruct them
3. Interactive Maintenance
  • Get relevant information to check the status of valve parts & repair them

Defining Interactions

Before diving into the specifics of the designing and implementation, it was important to understand what type of interactions fit well for such a use-case. The interactions were dependent on two specifics:

1. Navigation - Which type is relevant for the use-case
  • Physical egocentric navigation
2. Manipulation - What all the user can do
  • Selection
  • Positioning
  • Scaling
  • Rotation

System Concept Statement

Immersive Training is an augmented reality system which effectively realizes how the future of in-field assistance during maintenance of the hardware would look like. The system is primarily aimed at helping the beginner training personnel with their learning processes. Immersive Training also supports the learning of the hardware via interactive exploration of various elements. It has a support for speech recognition so that the user can use it hands-free. The system assists the user with audio as well as visual instructions by understanding the environmental context along with the current state of the user. By providing effective feedbacks and relevant information along with intuitive interaction techniques, Immersive Training system makes the learning process more fun, engaging, and useful.

Designing Solutions

Sketching

Intially we individually sketched out multiple ideas. Afterwards, we collaborated our ideas into forming 3 main scenarios for the storyboard. These scenarios are discussed in the upcoming section.

Storyboarding - User Flows

The three goals defined during the requirement analysis phase of the project helped us to translate them into three scenarios. The storyboards for these scenarios helped us to visualize how the system would behave under varying contexts.

  1. Idea generation: Individually
  2. Capturing ideas: Sketching
  3. Critiquing: Evaluate each other's ideas and proposals
  4. Synthesis: Analyze inputs from team members

Development of Prototype

The futuristic AR experience

In this section we discussed briefly details from some of the code modules. This section will provide a basic idea of how things were realised for the final prototype. The hardware used is a Meta2 AR glasses ad an external microphone. The development platform used was Unity 2018.1.8.

Marker detection using Vuforia

We planned to add object recognition feature to our system, to localize the target object in the real world and map the virtual elements on it. This was made possible by the SDK provided by Vuforia technologies. It uses computer vision technology to recognize and track planar images (Image Targets) and 3D objects in real time. To mimic the object recognition feature, we instantiate the 3D model of the valve at the location of the printed marker.

Canvas and Links

For our first scenario we needed the user to see names of the parts of the engine valve. These names had to be linked appropiately to the associated location on the main valve. To achieve this we provided visual linking by dynamically rendering lines from the labels to parts locations. Some of the parts of the vvalve were enclosed in the casing of the valve, hence we coded an X-Ray feature which allowed the user to see what's inside the casing. This feature can seen in the demo video of the project.

Sia - The Personal Virtual Assistant

For the first scenario, air touch interaction was sufficient for the user. However, as we moved forward to the other two scenarios, we realised that the user is usually pre-occupied with the task at hand. To redue the friciton caused by the interaction technique, we introduced a new modality - speech.

Using the Microsoft Windows speech library,the system was now able to respond user inputs fed through an external microphone. The system's virtual assistant is called 'Sia' and the user can trigger it using its name as a keyword.


Visual Instructions

Now we wanted the user to know what all he can achieve with our Immersive Training system. Hence, we added a tag-along canvas in the user's field of view which showed the possible commands that user can give at any given time.

Learnings

From exploring various data elicitation methods to taking a solution approach apart from designing just a smartphone app, I learned a lot. The biggest fear was to get the user requirements correctly. The answer lies in not only listening to what the user says but also observing what they do. I understood what it takes to successfully design a digital ecosystem around an already existing infrastructure. It's a fruitful practice to follow UX design lifecycle process at every checkpoint of the product development. Evaluating intermediate designs potentially saves a lot of time by providing a realistic idea of whether the finished product will solve the problem in reality or not.
